{MCM~中文论文~\LaTeX~模板} % 论文题目写在中括号内
% 下面空一行开始写正文

这里写文章摘要，第一段通常一句话概述。基本句型是：
本文根据~xxx~原理或理论，基于~xxx~算法，解决了~xxx~问题。

这里开始，一般按题目顺序依次简要介绍解决方案。一般一个问题一段，不要超过~3~段。
模板：
\begin{enumerate}
\item 概述子问题：该问题的本质/研究重点是~xxx~，可以用/基于~xxx~算法解决。
\item 算法的优化。最多三四句话阐述如何通过优化算法更好的解决了问题。
\item 三五句话专门陈述做的最有价值的部分。最好是原理/方法层面的创造性。
\end{enumerate}

\em{主要结果如下：}

小的错误，可以优化一下。如果没有做出结果，摘要里就不要提任何跟结果有关的字眼了。

从排版的角度来说，用一个醒目的字眼表明这里是结果。
我用自己定义的命令强调了``主要结果如下''几个字。命令格式可以看源码代码。

\em{主要创新点}

\begin{enumerate}
\item 按条列举，用~xxx~方法，更好的处理了~xxx~细节。
\item 一定要写对处理细节的创新，少而精。
让人看了就会认同，原理层面的，容易吹，看的人不信。写了没用。
\item 不管大小，总会找出一点创新性的。这个若是真没有，就省略了吧。
\end{enumerate}

最后是关键词，很多人不重视。
从心理学上说，关键词是可以低调的显摆，又不容易被戳穿的地方。
具体做法：回忆解题步骤，列举出每一步的最佳理论――不管自己有没有实现。
只要跟自己做的不矛盾就行。至于敢写多少，这就只能看长相了。

总体来说，摘要的一般讨论是，先简要介绍做的主要工作和结果。务必逻辑清晰，语言简洁。
然后适当的吹一下创新点。最后的关键词，没节操的吹牛就是了。
差点文章，吹了也没人在意。好的文章，吹一吹反而可以证明自己了解的多，理解的深入。
